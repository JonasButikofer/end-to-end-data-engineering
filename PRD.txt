with

orders as (

order_details as (

            object_construct(
                'SalesOrderDetailID',       sales_order_detail_id,

Here are a few more notes from the API docs on joining 
Data Characteristics
customer_id: Integer (range 11,000–30,118). Joins with stg_adventure_db__customers.
product_id: Integer (range 707–999). Joins with stg_adventure_db__products.
event_type: One of page_view, click, add_to_cart, purchase.
timestamp: ISO 8601 UTC. Covers the window from since to now (defaults to last 60 minutes).
session_id: Unique per browsing session. One customer may have multiple sessions.
Use the since query parameter for incremental extraction — only get events newer than your last pull. Event count scales with the time window (~50 per hour).

Events simulate realistic browsing funnels: page_view (50%) → click (25%) → add_to_cart (15%) → purchase (10%). Sessions contain 1–8 events.
